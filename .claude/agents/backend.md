## Code Generation Rules

For every module, generate the full layer stack:

```
<module>/
├── <Entity>.java                    # @Entity with all columns from schema.sql
├── <Entity>Repository.java          # JpaRepository + custom JPQL queries
├── <Entity>Service.java             # Business logic, validation, audit calls
├── <Entity>Controller.java          # @RestController, @RequestMapping, @PreAuthorize
├── dto/
│   ├── Create<Entity>Request.java   # @Valid request DTO
│   ├── Update<Entity>Request.java
│   └── <Entity>Response.java        # Response DTO (no password fields)
└── <Entity>Mapper.java              # MapStruct interface
```

## Module-Specific Requirements

### auth/
- `User.java` — implements `UserDetails`
- `JwtService.java` — generate/validate access + refresh tokens
- `AuthController.java` — POST /api/auth/login, /refresh, /logout
- `SecurityConfig.java` — stateless, CSRF disabled, CORS from `${FRONTEND_URL}`
- `JwtAuthFilter.java` — OncePerRequestFilter

### attendance/
- `AttendanceService.bulkMark(BulkAttendanceRequest)` — marks attendance for all workers on a site for a given date in a single transaction
- `BulkAttendanceRequest` DTO: `siteId`, `date`, `records: [{workerId, status, notes}]`
- Validation: cannot mark future dates, cannot overwrite attendance if already settled
- `amount_earned` computed server-side: PRESENT=full rate, HALF_DAY=rate/2, ABSENT=0

### wages/
- `WageService.computeLedger(workerId, siteId, fromDate, toDate)` — returns:
  ```
  {
    totalEarned: DECIMAL,
    totalWithdrawn: DECIMAL,
    netPayable: DECIMAL,
    attendanceDays: INT,
    halfDays: INT,
    absentDays: INT,
    withdrawals: [list]
  }
  ```
- `WageService.settleWorker(workerId, siteId, month, year)` — marks the period settled, triggers WhatsApp notification
- Ledger must never go negative — throw `InsufficientBalanceException` if withdrawal > net payable

### invoice/
- Auto-generate `invoice_number` on create: `INV-{YYYY}-{NNNN}` (zero-padded, per contractor)
- `InvoiceService.send(invoiceId)` — changes status to SENT, records `sent_at`, triggers WhatsApp to builder
- `InvoiceService.recordPayment(invoiceId, request)` — records payment, updates `amount_paid`, recalculates status (PARTIALLY_PAID if amount_paid < total, PAID if equal)
- `OverdueInvoiceScheduler` — `@Scheduled(cron = "0 0 8 * * *")` — daily 8am, flags SENT invoices past due_date as OVERDUE, sends WhatsApp to contractor

### progress/
- `ProgressController.getPublicSiteProgress(token)` — `GET /api/v1/sites/share/{token}` — no auth required, returns site progress for builder viewing
- Photo upload: multipart, stored in MinIO `siteops-progress` bucket, linked via attachments table

### files/
- `MinioService` — upload, download, presigned URL (15 min expiry), delete
- Buckets: `siteops-plans`, `siteops-progress`, `siteops-reports`, `siteops-attachments`
- `@PostConstruct` — ensure all buckets exist on startup

## REST Response Conventions

```java
// Success envelope
record ApiResponse<T>(T data, String message, int status) {
  static <T> ApiResponse<T> ok(T data) { return new ApiResponse<>(data, "success", 200); }
  static <T> ApiResponse<T> created(T data) { return new ApiResponse<>(data, "created", 201); }
}

// Pagination
record PageResponse<T>(List<T> content, int page, int size, long totalElements, int totalPages) {}

// Error
record ErrorResponse(String error, List<String> details, int status) {}
```

All paginated endpoints accept: `?page=0&size=20&sort=createdAt,desc`

## Error Handling

`GlobalExceptionHandler.java` must handle:
- `ResourceNotFoundException` → 404
- `MethodArgumentNotValidException` → 400 with field-level details
- `AccessDeniedException` → 403
- `InsufficientBalanceException` → 422 (wage withdrawal exceeds balance)
- `DuplicateAttendanceException` → 409 (attendance already marked)
- `BusinessRuleException` → 400 (general business rule violations)
- `Exception` (catch-all) → 500, log full stack trace

## Audit Logging

`AuditAspect.java`:
```java
@Aspect @Component
// @AfterReturning on all @Modifying-annotated service methods
// Captures: SecurityContext user, method name → action type
// Saves AuditLog entity: actor_id, action, entity_type, entity_id, change_set
```

`@Modifying` — custom annotation. Apply to every service method that mutates DB state.

## Security Annotations per Role

```java
// CONTRACTOR_ADMIN — full access
// SITE_SUPERVISOR — can mark attendance, add progress, view wages; cannot create/send invoices
// VIEWER — read-only

@PreAuthorize("hasAnyRole('CONTRACTOR_ADMIN','SITE_SUPERVISOR')")  // attendance marking
@PreAuthorize("hasRole('CONTRACTOR_ADMIN')")  // invoice creation/sending
@PreAuthorize("hasAnyRole('CONTRACTOR_ADMIN','SITE_SUPERVISOR','VIEWER')")  // reads
```
